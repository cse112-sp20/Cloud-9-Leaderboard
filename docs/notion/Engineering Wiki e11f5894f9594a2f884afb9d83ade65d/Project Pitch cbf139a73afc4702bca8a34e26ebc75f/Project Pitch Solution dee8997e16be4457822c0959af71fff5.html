Metrics to track

  • Keystrokes
    • vscode.workspace.onDidChangeTextDocument(
      • Event emitted whenever a doc in the editor is changed, unclear if this event is triggered when the user types, or when the user save the doc
      • Use this to determine how productive the user has been
  • Time
    • Simple, but allows a privacy option — maybe the user doesn't feel comfortable with us knowing when their files have changed.
    • Allow user to set a timer and turn off the timer
  • Settings
    • In settings, user can decide which metrics to track (both are an option)
    • We use their choice to determine their score function
    • Use the score to create a ranking of all users in the group
  • Google Chrome pages visited
    • Option to turn on/off for privacy concerns
    • Rewarded for visiting "research" (coding-relating) sites, punished for visiting entertainment sites

Workflow

  1. User opens repo their team is editing on VSCode
  1. User runs cloud 9 ext
    1. User logs in
    1. Tree view (or Webview?) is opened in sidebar
    1. Sees the current rankings in the group they're a part of
      1. Timer button also displayed for when they want to start working
    1. Can go to settings option to see details on their score function
      1. This could also be a command
  1. User starts working, to make their score go up

Explanation of the Solution

The primary problems we found with remote work were lack of motivation and procrastination. We blamed this in part due to the lack of interaction with others. So, as a solution, we offered a way to "gamify" work. This system would let users gain points the more they actively spend time in Visual Studio Code. As a result, this would be a Visual Studio Code Extension using WebView to let users see instructions, total points, and a breakdown. To encourage interaction with others, which would promote motivation to continue doing work, we will also add a leaderboard function to allow users to have a friendly competition with coworkers.

Though, we understand that not all of programming is solely spent in code editors. This is why we're also connecting a Google Chrome extension that would track user activity on the browser. The user would be penalized for entering entertainment or social media sites such as Netflix or Twitter that do not indicate productivity, but rewarded with points for entering research sites such as Oracle or StackOverflow. Since the primary target of this is programmers, we are keeping the view solely on Visual Studio Code, to encourage usage of the code editor.
