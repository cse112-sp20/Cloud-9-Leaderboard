UI Testing

VSCode Extension Tester (VSCode Test Framework for UI Testing)

We decided to select VSCoder Extension Tester for our UI testing because:

  • It is the most popular automated testing framework for JS
  • It is easy to set up with VSCode and a test runner script
  • Is easy to create tests for and installation is very simple
  • Documentation is very clear and there are great available examples

Here is a great source for more information on the functionality of this tool

How To Install:

  1. Download Node JS from their website
  1. Install Mocha npm install --global mocha
  1. Download Visual Studio Insiders from their website
  1. Run the following commands:
npm install --save-dev vscode-extension-tester mocha @types/mocha
npm install --save-dev chai @types/chai

How to Create and Run Tests:

  1. Add unit tests to test > suite > uittestts
  1. Run the following command:

`npm run ui-test'

Rules:

  • All UI tests must be written in the file uitest.ts (test > suite > uitest.ts)
  • For any tests that use await functionality, make sure the test includes the async keyword.
  • Make sure to increase the timeout time to 10,000 MS (this.timeout(10000);). UI tests can run at various speeds.
  • Make sure to allow the test to execute multiple times (this.retries();). UI tests can be very flaky and this helps prevent false negatives.
